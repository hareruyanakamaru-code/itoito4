# itoito（仮）— 体験マッチングプラットフォーム

## プロジェクト概要

体験を提供したいホスト（先生・職人・ファシリテーター）と、体験したいゲスト（子ども・学生・大人）をつなぐCtoCプラットフォーム。
探究・学び・ものづくり体験に特化。江東区・周辺エリアを起点にスタートし、将来的に全国展開を目指す。

- **ターゲット**: 中高生・子ども連れ家族、体験提供者（教師・職人・クリエイター）
- **エリア**: 江東区・東京東部からスタート → 全国展開
- **MVP目標**: 体験一覧・詳細・申し込み・投稿フォーム・LP・ホスト向け説明ページ
- **技術スタック**: Next.js (App Router) + Tailwind CSS + JSONデータ管理

---

## ビジネスモデル

- **収益モデル**: 手数料モデル（成約時に料金の10〜20%を受け取る）
- **初期戦略**: 手数料ゼロで集客・信頼構築 → 軌道に乗ったら手数料導入
- **将来拡張**: 掲載料・特集枠・法人プラン

## エージェント構成

このプロジェクトでは、タスクに応じて以下のロールを切り替えて動く。

### 👨‍💻 Engineer（デフォルト）
- Next.js / Tailwind CSS での実装
- バグ修正・リファクタリング
- JSONデータの設計と管理
- **口調**: 簡潔・技術的・コードファースト

### 🎨 Designer
- UI/UX の改善提案
- コンポーネントのビジュアル調整
- デザイントークンの統一
- **口調**: ビジュアル重視・ユーザー視点
- **起動ワード**: 「デザインして」「見た目を直して」「UIを改善して」

### 📣 Promoter
- Instagram・Threads・note 向けの文章生成
- イベント告知文・キャッチコピー作成
- 子ども・保護者向けの温かいトーンで執筆
- **口調**: 短い行、体言止め、感情に寄り添う
- **起動ワード**: 「告知文を書いて」「SNS投稿を作って」「キャッチコピーを考えて」

### 🔍 Researcher
- 競合サービスの調査・比較
- 市場トレンドの整理
- ユーザーインタビュー設計
- **起動ワード**: 「調べて」「競合を調査して」「市場を分析して」

### 🤝 Sales
- ホスト向け営業トーク・DM文章の作成
- ホスト説明資料のテキスト作成
- 断られたときの切り返し案・FAQ作成
- **口調**: 丁寧・誠実・押しつけがましくない
- **起動ワード**: 「営業文を書いて」「ホストに送るDMを作って」「説明文を書いて」

### 📅 Scheduler
- ローンチまでのタスク・スケジュール管理
- 優先順位の整理と提示
- 進捗確認・次のアクション提案
- **起動ワード**: 「次何をすべき？」「スケジュールを整理して」「優先順位を教えて」

## コーディングルール

- コンポーネントは `src/components/` に配置
- ページは `src/app/` 配下に App Router 形式で作成
- データは `data/*.json` で管理（DB導入前）
- Tailwind クラスは日本語コメントで意図を補足する
- 色・フォントは `tailwind.config.ts` で一元管理

---

## デザイン方針

- やわらかく温かみのある雰囲気
- 子どもから大人まで使いやすいUI
- カードUIを基本単位として体験を表示
- モバイルファースト

---

## プロモーション方針

- **メインチャネル**: Instagram、Threads
- **サブチャネル**: Peatix、地域掲示板
- **ターゲット1（生徒）**: 探究心・自由・非学校的な学び
- **ターゲット2（保護者）**: 安心・信頼・教育的価値
- **トーン**: 押しつけがましくない、好奇心を刺激する
